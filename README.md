# ProductCompare

เว็บเปรียบเทียบราคาต่อหน่วยแบบเร็วสำหรับใช้ตอนเดินซื้อของในเวอร์ชันเรียบง่าย กรอกชื่อสินค้า ปริมาณ และราคา แล้วระบบจะไฮไลต์ตัวเลือกที่คุ้มที่สุดทันที รองรับทั้งมือถือและเดสก์ท็อป

## ฟีเจอร์หลัก (เวอร์ชันปัจจุบัน)

- เริ่มต้นด้วยฟอร์มสินค้า 2 แถวทันที
- เพิ่มแถวสินค้าได้ไม่จำกัดด้วยปุ่ม เพิ่มรายการ
- ลบแถวสินค้าได้ทีละรายการ (แต่ต้องเหลืออย่างน้อย 2 แถวเสมอ)
- คำนวณราคาต่อหน่วยอัตโนมัติเมื่อกรอกข้อมูลครบ
- ไฮไลต์แถวที่ราคาต่อหน่วยต่ำสุด พร้อมเอฟเฟกต์ตอนมีผู้ชนะใหม่
- แสดงส่วนต่างราคาในแถวที่แพงกว่า
- แสดงข้อความเตือนเมื่อกรอกข้อมูลไม่ครบหรือค่าติดลบ/ศูนย์
- ปุ่มเคลียร์เพื่อรีเซ็ตหน้าจอกลับเป็น 2 แถวว่าง
- ปุ่มเปลี่ยนธีมแบบวน 5 ธีม: warm, ocean, forest, berry, midnight
- ใส่ cache-busting query string ให้ไฟล์ CSS/JS/manifest ทุกครั้งที่โหลดหน้า เพื่อเลี่ยงไฟล์ค้าง

## สิ่งที่ไม่มีในเวอร์ชันนี้

- ไม่มีระบบค้นหา
- ไม่มีการแชร์ลิงก์หรือแชร์รูป
- ไม่มีการบันทึกรายการหรือธีมข้ามการรีเฟรช
- ไม่มีการเรียงลำดับแถวอัตโนมัติ (แสดงตามลำดับที่ผู้ใช้กรอก)

## เทคโนโลยีที่ใช้

- Vanilla HTML + CSS + JavaScript (ไม่มี build step)
- Intl.NumberFormat สำหรับฟอร์แมตราคาเงินบาท
- Web Crypto API (crypto.randomUUID) สำหรับสร้าง id ของแต่ละแถว
- Web App Manifest สำหรับประสบการณ์ติดตั้งเป็นแอป

## การใช้งานในเครื่อง

เปิดไฟล์ index.html ด้วยเบราว์เซอร์ได้ทันที หรือใช้ local server เช่น Live Server ใน VS Code

ตัวอย่าง local server:

```bash
python3 -m http.server 5500
```

แล้วเปิด http://localhost:5500

## วิธีใช้งาน

1. กรอกชื่อสินค้า ปริมาณ และราคาในแต่ละแถว
2. กด เพิ่มรายการ เมื่อต้องการเพิ่มตัวเลือก
3. ดูผลราคาต่อหน่วยใต้แต่ละแถว
4. แถวที่คุ้มที่สุดจะถูกไฮไลต์อัตโนมัติ
5. ใช้ปุ่มลบเพื่อลบแถวที่ไม่ต้องการ
6. ใช้ปุ่มเคลียร์เพื่อล้างข้อมูลทั้งหมดและเริ่มใหม่

## พฤติกรรมการเก็บข้อมูล

- คีย์ Local Storage ที่ประกาศในโค้ด: product-compare-simple-rows-v1
- เมื่อโหลดหน้าใหม่ ระบบจะลบข้อมูลคีย์นี้และเริ่มจากข้อมูลว่างเสมอ
- ฟังก์ชัน persistRows เป็น no-op โดยตั้งใจ (ไม่บันทึกข้อมูล)

## Deploy บน GitHub Pages

โปรเจกต์นี้เป็น static web app จึง deploy ได้ทันทีจาก branch

1. Push โค้ดขึ้น GitHub branch main
2. ไปที่ Settings > Pages
3. เลือก Source: Deploy from a branch
4. เลือก Branch: main และโฟลเดอร์ / (root)
5. บันทึก แล้วรอระบบเผยแพร่

## โครงสร้างไฟล์

- index.html: โครงสร้าง UI หลัก และ logic สำหรับ cache-busting asset
- style.css: ธีม, responsive layout, และสไตล์ทั้งหมด
- app.js: ตรรกะจัดการแถวสินค้า, คำนวณราคาต่อหน่วย, ไฮไลต์แถวคุ้มสุด, และสลับธีม
- manifest.webmanifest: ข้อมูล web app manifest และไอคอน
- icon-192.svg: ไอคอนแอปขนาด 192
- icon-512.svg: ไอคอนแอปขนาด 512
