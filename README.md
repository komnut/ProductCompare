# ProductCompare

เว็บเปรียบเทียบราคาต่อหน่วยสำหรับใช้ตอนเดินซื้อของ กรอกชื่อสินค้า ราคา และปริมาณ แล้วระบบจะจัดอันดับว่าตัวเลือกไหนคุ้มที่สุดทันที รองรับทั้งมือถือและเดสก์ท็อป

## ฟีเจอร์หลัก

- เพิ่มรายการสินค้า (ชื่อสินค้า, ราคา, ปริมาณ) พร้อมตรวจสอบข้อมูลก่อนบันทึก
- คำนวณราคาต่อหน่วยอัตโนมัติ และเรียงผลจากคุ้มที่สุดไปแพงที่สุด
- ไฮไลต์ตัวเลือกที่ดีที่สุดด้วยสถานะ `⭐ คุ้มสุด`
- แสดงส่วนต่างแบบเข้าใจง่าย เช่น `แพงกว่า X บาท` ต่อแพ็ก
- ค้นหารายการตามชื่อสินค้า
- ลบทีละรายการ หรือ `ล้างทั้งหมด`
- ปุ่ม `ใส่ตัวอย่าง` เพื่อเติมข้อมูลทดสอบอย่างรวดเร็ว
- บันทึกรายการไว้ใน Local Storage (รีเฟรชหน้าแล้วข้อมูลไม่หาย)
- รองรับธีมสว่าง/มืด และจดจำธีมที่เลือก
- แชร์ผลเปรียบเทียบผ่าน `แชร์ลิงก์` (แนบข้อมูลลง URL เพื่อส่งต่อให้ผู้อื่นเปิดต่อได้)
- แชร์ผลเปรียบเทียบผ่าน `แชร์รูป` (สร้างภาพ PNG จากผลลัพธ์เพื่อแชร์หรือดาวน์โหลด)
- เมื่อเปิดลิงก์แชร์ ระบบจะโหลดข้อมูลให้อัตโนมัติแล้วลบพารามิเตอร์แชร์ออกจาก URL
- มี `manifest.webmanifest` สำหรับประสบการณ์แบบ web app
- ปิดการใช้งาน Service Worker cache และล้าง cache เดิมอัตโนมัติ เพื่อให้ได้ไฟล์ล่าสุดเสมอ

## เทคโนโลยีที่ใช้

- Vanilla HTML + CSS + JavaScript (ไม่มี build step)
- Canvas API สำหรับสร้างภาพแชร์
- Web Share API และ Clipboard API (เมื่อเบราว์เซอร์รองรับ)
- Local Storage สำหรับเก็บข้อมูลรายการและธีม

## การใช้งานในเครื่อง

เปิดไฟล์ `index.html` ด้วยเบราว์เซอร์ได้ทันที หรือใช้ local server เช่น Live Server ใน VS Code

ตัวอย่าง local server:

```bash
python3 -m http.server 5500
```

แล้วเปิด `http://localhost:5500`

## วิธีใช้งาน

1. กรอกชื่อสินค้า ราคา และปริมาณ
2. กด `เพิ่มรายการ`
3. ตารางจะแสดงราคาต่อหน่วยและเรียงรายการอัตโนมัติ
4. แถวที่คุ้มที่สุดจะแสดง `⭐ คุ้มสุด`
5. ใช้ช่องค้นหาเพื่อกรองรายการตามชื่อสินค้า
6. ใช้ปุ่ม `ลบ` เพื่อลบทีละรายการ หรือ `ล้างทั้งหมด` เพื่อล้างทั้งตาราง
7. ใช้ `แชร์ลิงก์` หรือ `แชร์รูป` เพื่อส่งผลลัพธ์ให้ผู้อื่น

## การเก็บข้อมูลและพฤติกรรมสำคัญ

- เก็บรายการสินค้าไว้ใน Local Storage key: `product-compare-items-v2`
- เก็บธีมไว้ใน Local Storage key: `product-compare-theme-v1`
- หากยังไม่เคยตั้งธีม แอปจะเริ่มจากค่าระบบ (`prefers-color-scheme`)
- ลิงก์แชร์ใช้พารามิเตอร์ `data` ใน URL (เข้ารหัสข้อมูลรายการ)
- การแชร์รูปจะใช้รายการที่เรียงแล้วสูงสุด 12 รายการแรก
- แอปไม่รองรับ offline cache แบบเดิมแล้ว (ตั้งใจให้โหลดไฟล์ล่าสุดเสมอ)

## Deploy บน GitHub Pages

โปรเจกต์นี้เป็น static web app จึง deploy ได้หลายวิธี

### วิธีง่ายสุด: Deploy จาก branch

1. Push โค้ดขึ้น GitHub branch `main`
2. ไปที่ `Settings > Pages`
3. เลือก `Source: Deploy from a branch`
4. เลือก `Branch: main` และโฟลเดอร์ `/ (root)`
5. บันทึก แล้วรอระบบเผยแพร่

### วิธีผ่าน GitHub Actions

หากต้องการใช้ GitHub Actions ให้เพิ่มไฟล์ workflow เอง (ปัจจุบันในโปรเจกต์นี้ยังไม่มีไฟล์ `.github/workflows/deploy.yml`)

## โครงสร้างไฟล์

- `index.html` โครงสร้าง UI หลัก
- `style.css` สไตล์, responsive layout, และ dark mode
- `app.js` ตรรกะฟอร์ม, คำนวณ, เรียงผล, แชร์ลิงก์/รูป, Local Storage, และธีม
- `manifest.webmanifest` ข้อมูล web app manifest
- `service-worker.js` โค้ดสำหรับยกเลิก Service Worker และล้าง cache เก่า
- `icon-192.svg` ไอคอนแอปขนาด 192
- `icon-512.svg` ไอคอนแอปขนาด 512
- `plan.md` แผนฟีเจอร์และ roadmap
